### Integration Issues

**Issue 1 — CONV-02: InquiryUrlWatcher client-side navigation** (Medium)

`InquiryUrlWatcher.tsx` uses `useEffect(fn, [])` which fires only on mount. When navigating from a case study page to `/?inquire` via `<Link>`, SiteShell stays mounted so InquiryUrlWatcher does not remount, and the modal does not open. Works correctly on hard page load.

**Fix:** Change `useEffect` deps from `[]` to `[searchParams]`, or make case study CTA use a button with `useShell().openInquiry()` instead of `Link href="/?inquire"`.

**Issue 2 — DEPL-02: Trailing slash inconsistency** (Low)

`data/case-studies.ts` uses `/case-studies/palo-alto-redwood-little-library` (no trailing slash) while `data/portfolio.ts` uses `/case-studies/palo-alto-redwood-little-library/` (with trailing slash). With `trailingSlash: true`, Nav links produce an extra 308 redirect.

**Fix:** Add trailing slashes to `data/case-studies.ts` hrefs.
